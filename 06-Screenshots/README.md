# 06 — Screenshots

Screenshots are organized to mirror the numbered folders above. Each one is referenced directly from the step file it belongs to, so the proof-of-work sits right next to the claim instead of in one undifferentiated dump.

```
06-Screenshots/
├── Environment-Setup/
├── Configuration/
├── Automation/
└── Testing-And-Troubleshooting/
```

Naming convention: `NN-Short-Description.png`, numbered in the order they appear within that phase's step files (e.g. `01-DC-NIC-Settings.png`, `02-ipconfig-DC.png`...). Each step file in this repo already has a `📸 Screenshot:` callout showing the exact filename and folder a given image should use — capture them in that order while working through the lab and they'll drop in cleanly.
